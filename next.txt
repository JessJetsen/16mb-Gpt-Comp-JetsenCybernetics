Branch-point objective

Take the current scratch trainer to a clean trainer-foundation branch point.

This branch is not competition-specific.
It is the generic high-fidelity trainer/control-surface branch that later splits into:

comp/ or records/... for course-specific competition logic
app-integration/ for SoftwareFactory model trainer import
current branch archived under /archive/learning/

Success means:

the important knobs are exposed,
they are grouped coherently,
they are layer-aware where appropriate,
they validate early,
they log/manifest clearly,
and the file stays comprehensible.

Do not overfit this branch to the golf model yet.

Design rules
Expose surfaces, do not hardcode policy.
The trainer branch should provide knobs and schedules, not encode a special strategy.
Layer awareness where it matters.
Any value that meaningfully differs by layer or role should have a path to per-layer override.
Prefer grouped controls over per-tensor chaos.
Roles first, layers second, individual tensor families third.
Every added surface must have:
env input
validation
log/manifest output
dashboard schema entry
No hidden magic.
If behavior changes at runtime, it must be inspectable in logs/manifest.
Do not blow past readability.
train_gpt.py is intended as a launch point and should stay under the project’s readability constraint.
